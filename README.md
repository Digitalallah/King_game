# King_game

Адаптированная web-версия карточной игры «Кинг» для Telegram Mini App.

## Что внутри

- Одиночная партия против трех ботов.
- Сетевая партия через Firebase Realtime Database: игроки автоматически входят в комнату по ссылке с `room_id`, видят лобби и синхронизируют ходы в реальном времени.
- Колода на 32 карты: 7, 8, 9, 10, J, Q, K, A четырех мастей.
- Полная последовательность из оригинальной DOS-версии: 7 штрафных контрактов и затем те же 7 контрактов на плюс.
- Правило оригинала для «червей», «Кинга» и «Ералаша»: нельзя первым ходом класть черву, пока на руке есть другая масть.
- Поддержка Telegram WebApp API: `ready`, `expand`, theme params, haptic feedback и Main Button.
- Адаптивный интерфейс для мобильного Telegram, desktop Telegram и браузера.

## Локальный запуск

Для одиночного режима можно открыть статические файлы:

```bash
python3 -m http.server 8080
```

Для сетевой игры заполните `FIREBASE_CONFIG` в `src/config.js` параметрами проекта Firebase с включенной Realtime Database. Локальный сервер нужен только для отдачи статических файлов:

```bash
node server.mjs
```

Откройте `http://localhost:8080`. Хозяин нажимает «Сеть» → «Пригласить» и отправляет созданную ссылку игроку. Игрок, открывший ссылку с `room_id`, автоматически подключается к Firebase-комнате; хост может начать партию после подключения минимум двух игроков.

## Оригинальные ресурсы

Бинарные файлы оригинальной DOS-игры и извлеченные PNG не хранятся в репозитории. Для локальной работы положите оригиналы в `assets/original/` и используйте скрипт `scripts/extract_original_assets.py`; этот путь добавлен в `.gitignore` и должен оставаться только ссылкой в коде или документации. Подробнее см. `docs/original-assets.md`.

## Деплой для Telegram Mini App

1. Разместите приложение на HTTPS-хостинге со статической отдачей файлов и доступом к Firebase Realtime Database.
2. В BotFather создайте Mini App и укажите URL на `index.html`.
3. Добавьте ссылку на Mini App в меню бота или кнопку Web App: `https://t.me/KingIgraBot/King`.
